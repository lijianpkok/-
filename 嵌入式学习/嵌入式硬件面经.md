#### ADC采样原理
**三个核心步骤：采样 → 量化 → 编码**

**采样（Sampling）**
- 定义：在连续模拟信号中，按固定时间间隔Ts取样本值
- 采样频率 fs = 1/Ts
- 奈奎斯特定理：fs ≥ 2×信号最高频率（fmax），才能无失真恢复原信号
- 例：音频采样 16kHz，可还原到 8kHz 以下信号

**量化（Quantization）**
- 定义：将采样得到的模拟值映射到有限的离散值
- 量化位数N，对应分辨率：ΔVFSR / 2^N（FSR=满量程范围）
- 量化误差：±ΔVFSR / (2×2^N)
- 例：12位ADC，FSR=3.3V，分辨率 = 3.3V / 4096 ≈ 0.8mV

**编码（Encoding）**
- 量化值转换为二进制代码（补码或偏移二进制）
- 输出数字信号供处理器读取

**关键指标**
- 分辨率（位数）：8/10/12/14/16/24bit
- 采样率（KSPS/MSPS）：取决于转换时间
- 准确度（INL/DNL误差）：指示实际性能与理想的偏差
- 功耗：重要指标，尤其在低功耗应用中

**实际应用考虑**
- 抗混叠滤波器：模拟滤波器 + 采样保持电路，防止高频噪声
- 驱动能力：ADC输入阻抗高，需要低阻抗驱动源
- 基准电压精度：直接影响量化精度

###  常见ADC架构，优缺点

**逐次逼近型**
内部的主要架构有DAC、比较器，使用二分法比较，直到DAC与输入电压压差小于1LSB，比较结束。
**优点：** 结构简单，功耗低。
**缺点：** 采样与转换速率慢，因为单次比较就需要DAC的RC输出与稳定建立时间、比较延时、逻辑延时，共需要比较n（位数）轮。
![[Pasted image 20260628101253.png|620]]
**FLASH型**
FLASH型ADC通过分压电阻，一次性生产一系列的比较电压，通过多个比较器与Vin进行同时比较;最终将比较结果编码输出。从编解码器我们就可以了解到(例如常见的3-8译码器和8-3编码器)，对于n位的FLASH型ADC输出，内部需要有2^n数量级个数的比较器。
**优点**：利用多个比较器来判别输入电压范围，单个时间周期完成转换。比较器数量越多，则分辨率越高。
**缺点**：分辨率有限，一般≤10bit（因为FLASH型ADC内部比较器规模为2^n级别，分辨率越高则芯片尺寸，成本暴涨）

![[Pasted image 20260628103321.png]]
**流水线型**
n位分辨率的流水线型ADC内部由n级1bit转换模块组成，每级转换模块内部构成为采样保持电路+1bitADC（即比较器）+1bitDAC+放大器 组成。
**举例一个4bit流水型ADC**
假设输入模拟电压Vin 0.4V，量程FSR为1V

**第一阶段（第1bit）：**
- 输入：Vin = 0.4V
- 比较电压：FSR/2 = 1V/2 = 0.5V
- 比较结果：0.4V < 0.5V → **输出 0**
- DAC输出：0.5V × 0 = 0V
- 残差放大：(0.4 - 0) × 2 = **0.8V** → 传给第二级

**第二阶段（第2bit）：**
- 输入：V1 = 0.8V
- 比较电压：0.5V（同上）
- 比较结果：0.8V > 0.5V → **输出 1**
- DAC输出：0.5V × 1 = 0.5V
- 残差放大：(0.8 - 0.5) × 2 = **0.6V** → 传给第三级

**第三阶段（第3bit）：**
- 输入：V2 = 0.6V
- 比较电压：0.5V
- 比较结果：0.6V > 0.5V → **输出 1**
- DAC输出：0.5V × 1 = 0.5V
- 残差放大：(0.6 - 0.5) × 2 = **0.2V** → 传给第四级

**第四阶段（第4bit）：**
- 输入：V3 = 0.2V
- 比较电压：0.5V
- 比较结果：0.2V < 0.5V → **输出 0**
- DAC输出：0.5V × 0 = 0V
- 最后一级无需放大

**最终结果：**
```
数字输出（从第1-4bit）：0110（二进制）
十进制值：0×8 + 1×4 + 1×2 + 0×1 = 6
验证：0.4V / 1V × 16(2^4) = 6.4 ≈ 6 ✓
```

**流水线型ADC的核心优势（体现在这个例子中）：**
- ✅ 每个阶段只需要1bit比较，简单快速
- ✅ 各级可以并行处理：第1级处理新输入时，第2级同时处理第1级的残差
- ✅ 与SAR ADC不同：SAR需要顺序比较16轮，流水线型仅需4个周期（管道延迟）
- ⚠️ 精度要求高：每级的DAC、放大器、比较器误差都会影响最终结果
**缺点**：采样延迟较长，对于n位分辨率的ADC，需要经过n级的转换、做差、放大过程。
![[Pasted image 20260628105217.png]]
同时，当残差V1从第一级输出给第二级时，此时第一级的工作已经完成，又可以接收新的模拟输入了;整个ADC架构也都是如此，像流水线一样工作。下表中stage1~4表示ADC的第一级到第四级，Time的T1，T2....Tn表示多个时间周期，S1~Sn表示一系列的模拟采样值，D1~Dn表示ADC完成的数字量转换。

**流水线型ADC的时间流水图：**

| 时间 | Stage1 | Stage2 | Stage3 | Stage4 | 输出结果 | 说明 |
|------|--------|--------|--------|--------|--------|------|
| **T1** | S1 进入 | 空闲 | 空闲 | 空闲 | — | 第一个采样值S1进入Stage1 |
| **T2** | S2 进入 | S1残差处理 | 空闲 | 空闲 | — | S2进入Stage1，S1残差转移到Stage2 |
| **T3** | S3 进入 | S2残差处理 | S1残差处理 | 空闲 | — | 每级都有任务，形成流水线 |
| **T4** | S4 进入 | S3残差处理 | S2残差处理 | **S1残差处理** | — | Stage4开始处理S1，即将输出 |
| **T5** | S5 进入 | S4残差处理 | S3残差处理 | S2残差处理 | **D1 = 0110** | 🎉 首个转换结果输出！ |
| **T6** | S6 进入 | S5残差处理 | S4残差处理 | S3残差处理 | **D2** | 每周期输出一个结果 |
| **T7** | S7 进入 | S6残差处理 | S5残差处理 | S4残差处理 | **D3** | 流水线满载运行 |
| **T8** | S8 进入 | S7残差处理 | S6残差处理 | S5残差处理 | **D4** | 持续输出 |

**关键对比 - 吞吐量分析：**

```
SAR ADC（12位例子）：
- 单个转换时间 = 36μs
- 输入 8 个采样：8 × 36μs = 288μs 才能全部完成

流水线型 ADC（4级）：
- 初始延迟 = 4 个时钟周期（管道充满）
- 每个采样处理时间 = 1 个时钟周期
- 输入 8 个采样：4 + 8 = 12 个周期（之后持续输出）
- 吞吐量提升：288μs → 12μs（24倍提升！）
```

**为什么能并行处理？**
```
T4 时刻的并行状态：
┌─ Stage1：处理新输入 S4 ───────┐
│                               │  4 级同时独立工作
├─ Stage2：处理 S3 的残差 ──────┤  产品级联关系，无依赖冲突
│                               │
├─ Stage3：处理 S2 的残差 ──────┤
│                               │
└─ Stage4：处理 S1 的残差 ──────┘  ← 即将输出第一个完整结果

特点：每一级只需关心前一级输入，不影响后续各级
```

**实际应用中的吞吐量：**
- 一旦流水线充满，每个时钟周期可以输出一个数字结果
- 采样率 = 工作频率 f_clk（理想情况下）
- 例：若 f_clk = 10MHz，则采样率 = 10Msps（百万样本/秒）
**∑-Δ型**
∑-Δ(Sigma Delta)ADC可以达到很高的精度，需要过采样、数字滤波消除量化噪声，从而实现高分辨率(最高可达24~32bit)。而这样做的代价是牺牲了采样速度，延迟变大，功耗也不低。基于这样的特性，∑-Δ(Sigma Delta)ADC在连续信号采集，高精度测量等领域有着广泛的应用。

**∑-Δ型工作原理：**
- 过采样：以远高于奈奎斯特频率的速率采样（通常高100~4000倍）
- 积分：将采样值累积（Σ）
- 反馈：与DAC反馈值做差（Δ）
- 数字滤波：消除高频量化噪声，提取有效信息

### 📊 四种ADC架构全面对比

| 对比维度 | **SAR（逐次逼近）** | **Flash（闪现）** | **Pipeline（流水线）** | **ΣΔ（Sigma-Delta）** |
|--------|----------|----------|----------|----------|
| **工作原理** | 二分法逐次比较，需n轮 | 2^n个比较器并行比较，单周期完成 | n级级联，每级处理1bit，管道运行 | 过采样+反馈+数字滤波 |
| **转换时间** | 慢（n个转换周期） | **极快**（1个周期） | 中等（n + 常数个周期） | 很慢（依赖过采样率） |
| **采样速率** | 低（KSPS～MSPS） | **超高**（MSPS～GSPS） | 高（MSPS） | 低（KSPS） |
| **分辨率** | 中（8~16bit） | 低（≤10bit） | 中高（10~16bit） | **超高**（24~32bit） |
| **精度** | 中等（INL/DNL：0.1~1LSB） | 低（因分辨率限制） | 中等（0.1~0.5LSB） | **极高**（精度可达0.001%） |
| **功耗** | **低** | 极高（比较器众多） | 中等 | 中等 |
| **芯片复杂度** | **简单** | 极复杂（指数级比较器） | 中等复杂 | 中等复杂（数字部分复杂） |
| **芯片面积** | **小** | 大（分辨率越高越大） | 中等 | 中等 |
| **成本** | **低** | 高 | 中 | 中高 |
| **噪声特性** | 普通 | 普通 | 普通 | **优秀**（噪声整形） |
| **对驱动源的要求** | 高 | 高 | 中 | 低（积分处理） |
| **应用场景** | 传感器、低功耗MCU | 高速示波器、视频 | 音频、中高速数据采集 | 高精度测量、音频、医疗 |

### 🎯 详细对比说明

**1️⃣ 速度角度**
```
Flash 👑 > Pipeline > SAR > ΣΔ

Flash：单周期完成，天生并行，速度无敌
Pipeline：初始延迟后，每周期1个结果
SAR：需要n次迭代（n=位数），最慢
ΣΔ：过采样倍数高，总时间最长
```

**2️⃣ 精度角度**
```
ΣΔ 👑 > Pipeline ≈ SAR > Flash

ΣΔ：通过过采样和数字滤波，噪声大幅降低
Pipeline/SAR：常规ADC精度
Flash：低分辨率限制精度
```

**3️⃣ 功耗角度**
```
SAR 👑 > Pipeline > ΣΔ ≈ Flash

SAR：只有1个比较器，功耗最低
Pipeline：多个比较器，但远少于Flash
ΣΔ/Flash：比较器众多或频繁操作，功耗高
```

**4️⃣ 集成度与成本**
```
SAR 👑 > Pipeline > ΣΔ > Flash

SAR：最简单，最容易集成，成本低
Flash：分辨率越高=比较器数越多，成本爆炸
Pipeline：折中方案
ΣΔ：需要精密模拟和数字电路
```

### 💡 如何选择ADC架构？

| 需求场景 | 推荐架构 | 原因 |
|--------|--------|------|
| **低功耗IoT传感器** | SAR | 功耗最低，集成简单 |
| **高速示波器、视频采集** | Flash | 需要极高速率 |
| **音频处理、中等采样率** | Pipeline | 速率足够，精度平衡 |
| **高精度测量、医疗设备** | ΣΔ | 需要超高精度 |
| **成本敏感产品** | SAR | 最便宜 |
| **多指标均衡需求** | Pipeline | 性能、功耗、成本均衡 |
